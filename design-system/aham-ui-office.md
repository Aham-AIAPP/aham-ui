# Aham UI · Office 导出规范（v5.1）

把 Aham 的屏幕语言迁移到导出的 **Word / PPT / Excel**。原则不变：**三层灰 + 蓝是点缀 + 文档风格表格（横线为主、表头靠字重不靠色块）+ 留白分隔**。Workbench 只管屏幕，这份补齐文档导出。

## 通用映射（先记这套）

| 角色 | 值 | 说明 |
|---|---|---|
| 正文 ink | `#262626` | 柔和近黑 |
| 次要 ink | `#6E6E6E` | 说明、表头副文字 |
| 三级 ink | `#9B9B9B` | 注释、页脚 |
| 面板灰 | `#F3F3F3` | Excel 表头底 / 区块底（极少） |
| 线灰 | `#E7E7E7` | 表格横线、分隔线 |
| 强调蓝 | `#336EE8` | 标题装饰条、关键数字、链接——**点缀，不铺底** |
| 强调蓝（深） | `#164EC3` | 蓝底上需要更深时 |

**字体（Office 可用优先）**
- 中文：**微软雅黑**（黑体 SimHei 备选）
- 西文：**Inter**（若未安装，退 Calibri / Segoe UI）——品牌一致建议在文档里嵌入 Inter
- 数字 / 单号：**Consolas** 或 JetBrains Mono（等宽，右对齐，千分位）
- 标题不用衬线；层级靠字号 + 加粗。**中文禁入等宽**（等宽只装数字）。

**铁规**：表格**横线为主、无竖线、无结构性底色**，表头/合计靠**加粗**不靠色块；数字列**右对齐等宽**；蓝**只**在标题装饰、关键数字、链接；**不要给整行整列铺蓝/铺灰**；留白分隔区块，少用框线。

---

## 一、Word（单据 / 报告 / 方案）

**页面**：A4，页边距上下 2.54cm、左右 2.5cm；正文雅黑/Inter 10.5pt（≈14px），行距 1.5。

**标题层级**（不用衬线，加粗拉开）
- 文档标题 H1：雅黑 22pt 加粗 `#262626`，下方一条 **`#336EE8` 2pt 短装饰线**（宽 ≈ 标题文字）。
- 一级 H2：16pt 加粗 `#262626`，段前 18pt。
- 二级 H3：13pt 加粗 `#6E6E6E`。
- 正文 10.5pt `#262626`；说明/注释 9pt `#9B9B9B`。

**表格（文档风格）**
- **只保留横线**：表头下一条 `#E7E7E7` 0.75pt、行间 `#E7E7E7` 0.5pt、**无竖线、无外框**。
- 表头：加粗 `#6E6E6E`，**不填色**。
- 合计行：加粗 `#262626`，上方一条 `#E7E7E7` 细线。
- 数字列右对齐、Consolas、千分位；分类列纯文本 `#6E6E6E`。
- 行高宽松（约 0.8cm），首末列贴内容边缘。

**引用 / 结论块**：左侧一条 `#336EE8` 2pt 竖线 + 浅缩进，**不填底色**。
**字段录入区**（如单据头）：标签 9pt `#6E6E6E` 在上、值 10.5pt `#262626` 在下；待补录用 `#8A7333`（琥珀）文字 +「待补录」，不填底。

---

## 二、PowerPoint（汇报 / 方案）

**尺寸**：16:9。**底永远纯白**，无渐变、无图片满铺、无色块墙。

**母版**
- 页脚一条 `#E7E7E7` 0.75pt 细线 + 右下页码 9pt `#9B9B9B`。
- 关键页可在左下放一条 `#336EE8` 2pt 短装饰条（≈3cm），作品牌点缀，**仅此一处蓝**。

**封面**：标题雅黑 32–40pt 加粗 `#262626`（左对齐，不居中堆叠）；副标题 14pt `#6E6E6E`；底部 `#336EE8` 短线。

**内容页**：标题 24pt 加粗 `#262626`；正文/要点 14–16pt `#262626`，行距 1.3。**要点用「•」+ 文字，不用图标、不用彩色块。** 编号用等宽 `01 / 02`。

**大数字页**：数字 Consolas 44–60pt 加粗 `#262626`（或关键一个数用 `#336EE8`）；标签 14pt `#9B9B9B`。一页一个核心数。

**图表**：柱/折线/环形——**灰阶 `#9B9B9B`/`#C8C8C8` + 一个 `#336EE8` 高亮当前项**，绝不多色。坐标轴线 `#E7E7E7`，标签 `#6E6E6E`。无 3D、无阴影、无渐变。

---

## 三、Excel（报表 / 成本核算）

**字体**：雅黑 11pt 正文；数字 Consolas 右对齐、千分位、负数前缀「−」。

**表头**：底 `#F3F3F3`（仅表头这一行可填浅灰）、文字加粗 `#262626`、下边框 `#E7E7E7` 细线。**正文区不填色**。
**网格**：去掉默认网格线，只保留 `#E7E7E7` 横向细线（行间）；**不画竖线**，列靠对齐和间距分。
**合计行**：加粗，上方 `#E7E7E7` 细线。
**冻结**：首行（表头）冻结。

**可编辑单元格（需人工填）**：浅琥珀底 `#F4F1E9` + `#8A7333` 文字提示，与只读区分。
**阈值/风险**：**只染文字色**（达标 `#5A7A60` / 预警 `#8A7333` / 超标 `#9E3D31`），**不填整格底色、不用红黄绿灯图标**。
**关键汇总数**：可用 `#336EE8` 文字强调（如总报价），全表**仅此类关键数用蓝**。

> 配套上传里有两个真实模板可参考样式：`Quotation Output Format_Costing Cal..xlsx`、`RFQ Output_Project Transfersheet.xlsx`。

## 落地提醒

- Inter / JetBrains Mono 若导出环境没装，用 Calibri / Consolas 兜底；要品牌一致就在 Office 文档里嵌入字体。
- 这套刻意**克制**：Office 默认爱给表头铺色、给单元格上红绿灯——按本规范一律改成"横线 + 加粗 + 文字染色"，才和屏幕端一致。

---

## 文本样式映射（对齐 DESIGN.md §1.3）
Word 段落样式 ← Aham 文本样式：标题1←Title(32) · 标题2←Heading(24) · 标题3←Subheading(20) · 正文←Body(14) · 说明文字←Caption(12)。
字体：西文 **Inter**(无则 Calibri) · 中文 **微软雅黑** · 数字/代码 **Consolas**(↔JetBrains Mono)。

## 文档不做暗色
Office 文档**固定亮色**(白底深字)，不随系统暗色——保证打印与分发一致。暗色仅用于网页 / 桌面 app。
