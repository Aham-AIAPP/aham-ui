# Aham UI 页面布局规范 — 审计 + Apple HIG 调研 + 规范草案

> 多智能体调研产出（19 个智能体并行：审计 / Apple HIG 调研 / 对抗式核验 / 综合）。
> **所有 px 数值为 Aham 自定取值；结构/顺序规则 ↔ Apple HIG。**

## 一、结论速览

Aham UI 在"原子层"（间距、布局宽度 token、控件高、断点、按钮 role、浮层 dismiss 行为）已相当完备，但在"页面骨架与模式层"普遍只有 `partial`/`weak` 覆盖：六大主题里没有一条是 `good`。最大系统性缺口是——**examples 里已经事实存在的约定（内容最大宽度、标题三段式、工具栏顺序、弹窗按钮顺序、预览交互）从未上升为 DESIGN.md 的成文规则**，导致 AI 消费时只能从示例反推、各页型取值漂移（max-width 在 420/560/760/780/920/1080 之间无规律）。Apple HIG 给出的恰恰是这一层的权威答案，且核验显示绝大多数调研条目准确——只有少数平台归属/定位措辞需要修正（已逐一标注）。建议：把 HIG 的"结构与顺序规则"成文，把"具体 px 数值"作为 Aham 自定 token 固定下来，二者分开。

| 主题 | 现状覆盖 | 最关键缺口 | 建议动作 |
|---|---|---|---|
| 整体居中 / 页面骨架 | partial | 内容最大宽度无标准值；居中 vs 左对齐无判定规则 | 新增「3.6 页面布局骨架」：定 max-width token、判定左对齐为默认 |
| 标题位置 | partial | 页眉区（面包屑+标题+主操作）总体结构与间距未成文 | 新增「页面页眉规范」：标题左上对齐、主操作右上、间距固定 |
| 搜索与筛选 | partial | 工具栏左右顺序、筛选 chips/结果计数/清除位置未定 | 新增「搜索与筛选布局」：三段式工具栏 + chips/计数规范 |
| 操作按钮位置 | partial | 确认/取消左右顺序、默认按钮、Return/Esc 绑定未成文 | 补全「3.3 动作主次」+「2.7 浮层」：锁死 macOS 按钮顺序 |
| 弹窗操作 | partial | modal/alert/sheet/popover 选型边界、未保存确认未成文 | 补全「2.7 浮层」：组件选型决策树 + 未保存确认硬规则 |
| 预览操作 | weak | 整套 Quick Look 式预览模式缺失（触发/翻页/缩放/预览vs打开） | 新增「预览模式 Preview」整节 |

## 二、逐主题：现状 → Apple 怎么做 → 差距

### 2.1 整体居中 / 页面布局骨架

**现有规范（DESIGN.md / tokens / css）**
- 1.4 布局与间距：间距 4 基 `4/8/12/16/24/32/48/64/96`；布局 `rail 60 / sidebar 264 / rightbar 340 / topbar 52`；断点 `≤860 折叠/转单列，≤380 压窄`。
- 1.4：网页"定义内容最大宽度 + 响应外边距"，但未给值；本地化变长不截断；空间不足时规定折叠/overflow/截断优先级。
- 3.4：卡片 padding 16–24；嵌套 padding 递减 24→16→8。
- aham-ui.css：`.shell{display:flex;height:100vh;overflow:hidden;}` 三栏骨架已实现。
- tokens.json：layout 段含四宽度；breakpoint `sm380/md860/lg1280`。

**examples 事实约定**
- max-width 各页型不一：patterns 1080、composition 560、navigation 760、data 780、charts 920、登录页 420——全部 `margin:0 auto` 居中，但未成文。
- 页面 padding 实测 20–24px；响应式 `@media(max-width:760px)` 三栏→单列折叠已在示范中实现。

**Apple HIG 关键规则（均已核验属实，除特别标注）**
- **默认左对齐、按阅读顺序排（top→bottom、leading→trailing），最重要内容放左上；不要默认整体居中**（Layout，居中是 visionOS 特例，与 Aham 无关）。✅
- **用系统 layout guide 套标准外边距，并用可读宽度上限限制正文行宽**——HIG 不给固定 px，由系统/模板决定（Layout）。✅
- **内容铺满到窗口边缘**，背景/可滚动区延伸到边；控件层（sidebar/toolbar）浮在内容之上；内容不满宽时用 background extension view 在 sidebar/inspector 两侧补足（Layout、Sidebars）。✅
- **窄屏"晚折叠"**：先按全尺寸布局设计，放不下才切 compact，优先隐藏最右的 inspector/第三列（Layout）。⚠️ 核验提示：该 defer-compact 指南原文位于 **iPadOS 小节**，原文仅称其缩放行为"类似 macOS"，并未明确扩展到 macOS——作为通则借鉴可以，勿宣称"HIG 明确说 macOS 同理"。
- **不要把控件/关键信息放窗口底部或 sidebar 底部**；不在 Mac 顶部摄像头区放内容（Layout/Windows/Sidebars）。✅
- 安全区 = 不被 toolbar/tab bar 遮挡的区域，布局须尊重并随尺寸动态重排（Layout）。✅
- 栅格：列数由内容宽度+间距自动决定，间距保持一致（Layout）。✅

**差距**
1. 内容最大宽度无标准值/推导规则。
2. 页面级 gutter/左右外边距无规范（示范用 20–24px）。
3. 居中 vs 左对齐无决策方法——HIG 明确默认左对齐，Aham 示例却普遍居中，方向相反。
4. shell 三栏/双栏/全宽的组合方案未文档化。
5. 窄屏折叠无对应 CSS class/模式。
6. 缺完整页面布局成品 HTML（patterns.html 是示例代码非规范）。

### 2.2 标题位置

**现有规范**
- 1.3：Display 44 / Title 32 / Heading 24 / Subheading 20。
- 2.3：`.navbar` 三段式 返回`.back`(蓝) / `.nav-title` / `.nav-actions`。
- 3.2：4 基网格；表单 label/控件/说明左缘成线；数字右对齐。
- 3.4：侧栏选中↔内容标题联动。
- css：`.navbar` 与 `.titlebar` 均 `height:52px` flex；`.nav-actions{margin-left:auto}`。

**examples 事实约定**
- patterns 详情页：面包屑→12px→标题(text-heading 24px)左对齐 + 页面操作同行居右→4px→页签。
- navigation：navbar 三段式，标题 `.nav-title`(15px)。
- 登录页：大标题 text-heading 居中 + 副标题居中。

**Apple HIG 关键规则**
- **macOS 窗口标题在窗口顶部框架（title bar），可与工具栏整合、与控件内联同行**（Toolbars）。✅
- **内容区/视图标题默认贴工具栏 leading，紧跟返回按钮与侧栏开关之后**（Toolbars）。✅
- 视图标题也可放工具栏中部，但中部会随窗口缩小折叠进 overflow，不保证常驻（Toolbars）。✅
- **标题简短 <15 字符**；不要用 app 名当窗口标题（Toolbars / The menu bar）。✅
- **最重要信息（含主标题）放窗口顶部+leading（左上），不居中**（Layout）。✅
- 区分两类标题：navbar/title-bar 标题（轻量、回答"在哪"）vs 内容区大标题（醒目、回答"这块内容是什么"）。✅
- 主操作放工具栏 trailing 且只一个，用 .prominent 着色（Toolbars）。✅
- ⚠️ 核验否定项：① "macOS sheet 锚定在父窗口顶部"无原文支撑（原文只说 floats on top + 父窗变暗）。② "窗口默认 1280×720 pt、设最小/最大尺寸"实为 **visionOS 段**，不是 macOS 默认。③ "窗口控件（红黄绿）不能与工具栏前缘项重叠"出自 **Windows 页 iPadOS（windowed）段**，非 macOS 信号灯条款。

**差距**
1. 页面级标题与副标题是否同行、间距标准未定。
2. 面包屑↔标题距离未成文（示范 12px）。
3. 标题区（面包屑+标题+操作）总体上下 padding 未定。
4. navbar `.nav-title` 与内容区 `.text-heading/.text-title` 分工未规定。
5. 主操作与标题同行的对齐方式（baseline/center）未定。
6. 窄屏标题与操作是否折行及优先级未定。
7. 标题下分隔线是否强制、间距未定。

### 2.3 搜索与筛选

**现有规范**
- 2.3：`.search` = input-wrap(放大镜前缀 + `.clear`) + 建议下拉 `.search-suggest`。
- 2.1：`.toolbar`(溢出 `.overflow` → More 菜单)。
- 3：一组一 primary；主操作在右/末位；可见按钮 ≤3–4 多则收菜单。
- css：`.toolbar{display:flex;align-items:center;gap:10px;flex-wrap:wrap;padding:14px 20px;}`。

**examples 事实约定**
- patterns 工具栏：搜索框(200px) + 下拉筛选(120px) + 筛选 tag + 右侧操作(导出 flex:1 推到右端)，筛选 chips 用 `.tag` 含 × 移除。

**Apple HIG 关键规则（本主题核验全部准确）**
- **macOS/iPadOS：搜索框默认放工具栏 trailing（右）侧**（Search fields）。✅
- **搜索用作过滤当前视图时，放在被过滤内容旁/上方**（inline 或 sidebar 顶部）；滚动时考虑钉顶（Search fields）。✅
- **区分全局搜索 vs 局部过滤**："search acts as a filter on the current view"（Searching）。✅
- **清晰展示搜索范围**：用占位文字、scope bar 或标题标明（Searching）。✅
- **筛选 chips 用 token 放搜索框内**（可编辑），更复杂范围切换用 **scope bar 放结果内容区顶部**；默认从最宽范围开始（Search fields）。✅
- 结果最相关优先——**HIG 未规定独立"结果计数"位置**，计数通常并入标题/视图头部（Search fields）。✅
- **工具栏三段式**：leading（侧栏开关/返回+标题）— center（常用控件，可折叠进 overflow）— trailing（常驻重要项、inspector 开关、可选搜索框、More、唯一主操作）（Toolbars）。✅
- 溢出由系统自动处理，**不手动加溢出菜单**（Toolbars）。✅
- 结果计数/清除/关键筛选不要放窗口或 sidebar 底部（Layout/Windows/Sidebars）。✅

**差距**
1. 搜索框在三栏中的层级位置（顶栏 vs 工具栏 vs 内容区）未明确。
2. 筛选器与搜索框左右顺序未约定。
3. 筛选 chips 与"清除全部"位置未规定。
4. 结果计数文案位置/样式未规定。
5. 窄屏工具栏折叠/溢出细则未定。
6. 搜索框最小/最大宽度未约束。
7. 搜索建议下拉与筛选下拉同时出现的 z-index 处理未定。

### 2.4 操作按钮位置

**现有规范**
- 3.3：一组一 primary；层级 primary>secondary>ghost>danger；**主操作在右/末位**；可见按钮 ≤3–4；按钮间距 8–12。
- 2.1：role `normal/primary/destructive`；destructive 且不可撤销必须二次确认。
- css：`.modal__foot` 与 `.dialog .dlg-foot` 均 `justify-content:flex-end;gap:8px`；`.doc-table td.row-actions{text-align:right}`。

**examples 事实约定**
- 列表页 titlebar：primary"新建"右侧（space-between）。详情页：分享/导出 secondary 靠右。对话框 dlg-foot：取消(ghost)左、删除(danger)右。登录页：单一 primary 全宽。

**Apple HIG 关键规则**
- **弹窗默认/确认按钮放按钮行 trailing（右下），Cancel 放其左（leading）**；竖排默认在顶、Cancel 在底（Alerts）。✅
- **绝不把 primary（默认）角色给破坏性按钮**；destructive 用 system red（Buttons）。✅
- 破坏性样式分情况：用户主动发起（Empty Trash）不套 destructive 样式、保留 Return 确认便利；非用户本意才用红（Alerts）。✅
- 四种 role：Normal/Primary/Cancel/Destructive；用样式而非尺寸区分首选项；**每视图最多 1–2 个 prominent 按钮**（Buttons）。✅
- 工具栏只一个 primary、放 trailing、.prominent 着色；分组 ≤3；优先全放工具栏，确需才加 More（Toolbars）。✅
- 行尾放行内附件：info 按钮=看详情、disclosure indicator(chevron)=层级导航，**二者不可混用**（Lists and tables）。✅
- 按钮命中区 ≥44×44 pt；标题用动词、避免 OK；push 按钮打开新窗口/视图时标题加省略号(…)（Buttons）。✅
- ⚠️ 核验否定项：Mail "New Message" 例出自 **Toolbars iOS/iPadOS 的 scrolling toolbar button 小节**，HIG 把它定性为"important action that **isn't** a primary app function"，与 trailing primary action 是两套机制。页面级主操作放右上的结论本身成立（由 Toolbars trailing + Layout 左上阅读顺序支撑），但勿用此例作论据。

**差距**
1. 页面级主操作"必须右上角"无成文条文。
2. 工具栏多按钮排布优先级/进 overflow 逻辑未成文。
3. 确认/取消左右顺序无文字规范（示范取消左、确认右，但未写死）。
4. 默认按钮（Return/Esc 绑定）未定义。
5. 收菜单的何时/如何细则缺失。
6. 行内操作(edit/view/delete)顺序未规定。
7. modal footer 换行对齐未定。

### 2.5 弹窗操作

**现有规范**
- 2.7：`.modal`(`__head/__title/__body/__foot`) / `.dialog`(确认弹窗) / `.drawer`；dismiss：modal 用按钮区、popover/tooltip 点外即消、Esc 关非破坏浮层、有未保存内容关闭前确认、sheet 在 macOS 总是模态；仅浮层有阴影、同心圆角。
- css：`.dialog{max-width:380px;padding:24px}`；footer flex-end gap:8px。
- js：`[data-open]/[data-close]/.scrim` 点击关闭；Esc 关闭所有可见浮层。

**examples 事实约定**
- overlays：确认弹窗 取消(左)/删除(danger,右)；popover 仅展示提示无操作按钮。

**Apple HIG 关键规则（本主题核验全部准确）**
- **按钮顺序锁死 macOS**：默认/确认右下（trailing）、Cancel 在其左；默认按钮=primary role、绑 Return；破坏性按钮不设默认；想强制阅读时可不设任何默认（Alerts/Buttons）。✅
- 避免 OK 作默认；用具体动词；Cancel 永远叫"Cancel"（Alerts）。✅
- Alert 结构：必填标题 + 可选说明 + 至多三按钮；慎用 alert（不为常见可撤销操作弹、不在启动时弹）（Alerts）。✅
- **Modal 仅在明确收益时用**；一次只一个模态视图，绝不同时显示两个 alert/级联 popover（Modality/Popovers）。✅
- **关闭方式遵循平台**：macOS 按钮在主内容区、Esc 或 Command-Period 关闭（Modality/Alerts）。✅
- **未保存确认硬规则**：任何关闭会丢失用户内容（含下滑/Esc/点外部），先解释并给保存/放弃/取消出路（Modality）。✅
- 组件选型：关键信息/警告→alert（**绝不用 popover 报警告**）；输入/小任务→sheet（macOS 模态、浮于父窗、父窗变暗）；少量临时→popover（箭头指源、点外即关、**非模态自动关闭时必须保存**）；反复输入看结果→panel 而非 sheet（Sheets/Popovers/Modality）。✅
- Sheet 按钮：Done 必配 Cancel 或 Back（Sheets）。✅
- Action sheet：破坏选项置顶用 destructive 样式、Cancel 置底——注意这与 Alert"默认在 trailing"是不同组件不同约定（Action sheets）。✅

**差距**
1. modal vs alert vs sheet vs drawer 选型边界未规定。
2. 确认/取消顺序未明文。
3. 打开 modal 时默认焦点落点未说。
4. 关闭方式矩阵（modal 是否 Esc/是否显 X/popover 是否有 X）未成文。
5. "保存/放弃/取消"三按钮语义与顺序未定。
6. 标题/正文强制性、最小结构未定。
7. `.modal` 宽度范围未定（仅 `.dialog` 380px）。
8. 响应式按钮换行/堆叠未定。
9. 焦点陷阱/关闭后焦点归还、aria-modal/aria-labelledby 未声明。

### 2.6 预览操作

**现有规范（weak）**
- 2.6：`.image-box` / `.share-panel` / `.edit-menu`。2.7：`.modal/.dialog/.drawer`。2.3：navbar 三段式。
- css：`.image-box{...aspect-ratio:16/9}`；`.share-panel{grid-template-columns:repeat(4,1fr)}`。

**examples 事实约定**
- status-viz：图片容器占位/1:1/16:9/加载中，但**无预览交互**；分享面板 4 列导出。无任何"点图进预览"的完整流程示范。

**Apple HIG 关键规则（本主题核验全部准确）**
- **Quick Look 本质**：在 app 内预览（即使 app 打不开该文件），可做轻量交互（看图/听音/markup/旋转缩放 3D）——预览=轻量查看+少量交互，**不是完整编辑会话**（File management）。✅
- **预览 vs 打开**：复杂/长流程编辑用独立窗口或全屏，不用预览面板/sheet（Sheets）。✅
- **inspector** 优先用 panel 或 split view pane，浮于主窗，**跟随选中项实时刷新**；内容固定的 Info 用普通 window；inspector 内优先 slider/stepper（Panels）。✅
- panel 短标题（名词）；app 激活时所有 panel 置前、失活全隐藏；一般无最小化按钮（Panels）。⚠️ 微修：显隐命令原文说可放 **Window 菜单**。
- **HUD 深色半透明面板仅用于沉浸式媒体预览**；常规检查器用标准浅色面板（Panels）。✅
- **列视图：选中项无子项时，最后一列显示预览 + 创建日期/修改日期/文件类型/大小**四项元数据；列视图仅 macOS（Column views）。✅
- **全屏预览**：可临时隐藏工具栏/导航，用熟悉动作恢复（点击/下滑/光标移顶部）；退出由用户决定不自动退出；用系统全屏机制（⌃⌘F）（Going full screen）。✅
- image view 纯展示通常不可交互；可点击用 image button、可编辑用 image well；可 stretch/scale/size-to-fit/pin（Image views）。✅
- 预览内轻量编辑（markup）应自动保存（File management）。✅

**差距**
1. Quick Look 式（空格）预览交互/布局未规定。
2. 预览面板/inspector 布局结构与尺寸未规定。
3. 上一张/下一张导航位置、样式、快捷键未规定。
4. 缩放/适应行为（contain/cover/zoom）未规定。
5. 预览 vs 打开区别（浮层 vs 全屏 vs 新窗口）未规定。

## 三、规范草案（可直接并入 DESIGN.md）

> 写法遵循 Aham 既有风格（短句、`反引号` 标类名、↔Apple 标注对应、数值自定）。**所有 px 数值为 Aham 自定取值，结构/顺序规则 ↔ Apple HIG。**

### 新增「3.6 页面布局骨架」

**内容最大宽度（Aham 自定值，↔Apple"可读宽度上限"）**
- `--content-max: 1080px`（宽页/列表/详情主内容上限，对应 examples patterns 既有值）。
- `--content-max-narrow: 720px`（长正文/阅读型，行宽可读上限）。
- `--content-max-form: 560px`（表单/组合卡片，对应 composition）。
- `--content-max-auth: 420px`（登录/单任务窄屏）。
- 依据：HIG「restrict the width of text for optimal readability」——正文不铺满全宽；具体值 Aham 自定，统一为上述四档，杜绝随手取值。

**居中 vs 左对齐判定（↔Apple Layout，默认左对齐）**
- 默认：内容区**左上对齐**，按阅读顺序 top→leading 排，最重要信息放左上。
- 居中仅限：单任务全屏页（登录/空状态/向导），用 `--content-max-auth/form` + `margin:0 auto`。
- 列表/详情/数据/设置等主工作区：**左对齐**，内容容器 `max-width` 取上述档位但**靠 leading 不居中**。背景/可滚动区延伸到边缘（↔Apple"extend content to fill"）。

**外边距 / gutter（Aham 自定）**
- 页面内容区左右 gutter：`--s5`(24px) 桌面；`≤md(860)` 收为 `--s4`(16px)；`≤sm(380)` 收为 `--s3`(12px)。
- 顶部 padding `--s5`、底部 `--s6`(32px)。底部**不放控件/关键信息**（↔Apple"avoid bottom of window")。

**页面三层结构（新成文）**
1. **页眉区** `.page-header`：面包屑/返回 + 标题 + 页面级主操作。
2. **工具区** `.page-toolbar`：搜索/筛选/视图切换。
3. **内容区** `.page-content`：受 `--content-max` 约束、左对齐。

**shell 组合方案（成文）**
- 三栏：`rail60 + sidebar264 + 内容 + rightbar340`。`≤lg(1280)` 优先收 `rightbar`(inspector)，`≤md(860)` 收 `sidebar`（可折叠，**不默认隐藏**）。"晚折叠"：先全栏，放不下才折。

### 新增「页面页眉规范」（并入 2.3 或 3.6）

- **标题位置**：内容区主标题 `.page-title` 用 `text-heading`(24) 或 `text-title`(32)，**左对齐、置页眉区左上**。
- **navbar 标题 vs 内容标题分工**：`.nav-title`(15px) = 窗口/位置标识，简短 **<15 字符**；`.page-title` = 内容主标题，醒目。二者不重复，不印 app 名当标题。
- **面包屑/返回**：`.breadcrumb`/`.back`(蓝) 置标题**上方**，间距 `--s3`(12px)。
- **副标题**：`.page-subtitle` 用 `text-body`/`ink-2`，置主标题**下方** `--s2`(8px)，不与标题同行。
- **页面级主操作**：放页眉区**右端（trailing）**，`justify-content:space-between`；**全页只一个 primary**(单蓝 prominent)。
- **页眉区总边距**：上下 `--s5`(24px)；标题与下方页签/分隔线间距 `--s4`(16px)。
- **窄屏**：`≤md` 主操作可降为图标按钮或进 More；面包屑可截断中段。

### 新增「搜索与筛选布局」（补 2.3）

- **搜索框落位**：全局搜索 → 工具区/topbar **trailing（右）**；局部过滤 → 该列表**上方**或 sidebar 顶部，滚动时钉顶；同一控件不承担两种语义。
- **工具栏三段式从左到右**：leading（视图切换/返回）— center（次要控件）— trailing（搜索框 + 唯一主操作）。
- **筛选 chips**：用 `.tag`(token) 放搜索框内或紧贴其下，含 × 移除；复杂范围切换用 scope bar 放结果区顶部；默认最宽范围。
- **结果计数**：左对齐并入内容区头部，用 `text-xs`/`ink-2`；**不放底部**。
- **清除全部**：放 chips 行尾（ghost/链接样式）；单次输入清除用搜索框内 `.clear`。
- **搜索框宽度**：`min 200px / max 360px`；`≤md` 折为图标触发。

### 补全「3.3 动作主次」与「2.7 浮层」：弹窗按钮顺序

**macOS 弹窗按钮顺序（硬规则，↔Apple Alerts）**
- 水平排：**Cancel 在左（leading）、默认/确认在右（trailing）**；竖排：默认在顶、Cancel 在底。
- `.modal__foot`/`.dlg-foot` 保持 `justify-content:flex-end`，按钮 DOM 顺序 = 取消在前、确认在后（视觉确认在右）。
- **默认按钮**绑 `Return`（primary，单蓝实心）；`Esc` / `⌘.` 关闭 = Cancel。
- **破坏性操作**：删除/清空用 danger；用户主动发起→确认可作默认、可 Return；非用户本意→红色 destructive + **不设默认** + 必配 Cancel。
- **绝不把 primary（蓝色默认）给破坏按钮**；**绝不把 Cancel 设为默认**。
- 按钮文案用动词；纯信息提示才用 OK。

**Web 与 macOS 差异**
- Web 常见"确认在左"——**Aham 一律按 macOS（确认在右）**，桌面与网页保持一致。

**默认焦点**
- 打开弹窗焦点落默认按钮；破坏性非主动场景焦点落 Cancel。焦点陷阱：Tab 在弹窗内循环；关闭后焦点归还触发元素。

**保存/放弃/取消三按钮（未保存确认）**
- 顺序（左→右）：取消 / 放弃(danger) / 保存(primary 默认)。任何关闭会丢内容（含 Esc/点外/下滑）必先弹此确认。

**组件选型决策树（补 2.7）**
- 关键信息/警告 → `.dialog`(alert)。**警告绝不用 popover**。
- 要求输入/小任务 → sheet（macOS 模态、浮于父窗、父窗变暗）。
- 少量临时信息/快捷设置 → `.popover`（箭头指源、点外即关；非模态自动关闭必须保存）。
- 反复输入看结果 → panel（非 sheet）。一次只一个：不叠两个 alert、不级联 popover。

`.modal` 宽度（Aham 自定）：`min 360px / 默认 480px / max 640px`；`≤sm` 全宽留 `--s4` 边距，按钮可堆叠垂直（确认在上、取消在下）。

### 新增「2.8 预览模式 Preview」（整套，↔Apple Quick Look/Panels/Going full screen）

**触发与语义**
- 列表/网格条目支持 **空格 = 轻量预览**、**双击/回车 = 打开（进编辑/全屏工作区）**。
- 预览 = 临时、单焦点、只读或轻量交互；打开 = 完整会话。即使非原生格式也应能预览。

**预览容器（Aham 自定）**
- 默认 **居中浮层** `.preview`（媒体类）或 **全屏** `.preview--full`（沉浸看图/幻灯片）。
- 浮层尺寸：`max-width: min(90vw, 1080px)`，`max-height: 90vh`，图片 `object-fit:contain`。
- 全屏：可临时隐藏工具栏，光标移顶部/点击恢复。

**翻页（上一张/下一张）**
- 左右箭头按钮置浮层左右两侧垂直居中；快捷键 `←/→`；多文件显示计数器 `3 / 12` 置顶部居中或底部。

**缩放/适应**
- 默认 fit（contain）；支持 `+/-` 或滚轮缩放、`0` 复位；放大态光标 `grab/grabbing`。

**预览内操作栏**
- 顶部 trailing 放分享/导出/删除（图标）；删除遵循弹窗规则（可撤销则 Undo 不弹 alert）。

**inspector/元数据（↔Apple Column views）**
- 预览侧栏（或选中无子项时）展示：**创建日期 / 修改日期 / 文件类型 / 大小** 四项；inspector 随选中项实时刷新，用 panel/`rightbar` 实现。

**关闭**
- `Esc` 或右上 `×`(浮层) / 退出全屏由用户决定；切走再回保持原位（媒体自动暂停）。
- 有未保存 markup → 关闭前确认。

**HUD 限制**
- 深色半透明 HUD 仅用于全屏媒体预览；常规 inspector 一律浅色标准面板。

## 四、如何保证统一（机制）

**1. tokens.json 增项**
- 新增 `content` 段：`max: 1080 / max-narrow: 720 / max-form: 560 / max-auth: 420`。
- 新增 `gutter` 段：`desktop: 24 / md: 16 / sm: 12`。
- 新增 `dialog`/`modal` 宽度：`dialog: 380 / modal-min: 360 / modal-default: 480 / modal-max: 640`。
- 新增 `preview`：`overlay-max-w: 1080 / overlay-max-h-vh: 90`。
- 新增语义键：`button-order: cancel-leading,confirm-trailing`、`default-key: Return`、`dismiss-key: Esc,Cmd+Period`，供 AI 直接消费。

**2. AGENTS.md 消费顺序**
- 明确七层读取序：tokens → DESIGN.md(规则) → aham-ui.css(实现) → examples(成品参照) → 校验层。
- 标注"结构/顺序规则来自 HIG（不可改），px 数值来自 Aham token（可调）"——避免 AI 把 HIG 结构当可选项。

**3. examples 需补的成品页**
- `page-shell.html`：三栏/双栏/全宽 + 三层结构 + 响应式折叠 class。
- `page-header.html`：面包屑+标题+主操作右上的标准页眉。
- `search-filter.html`：工具栏三段式 + chips + 计数 + 清除完整流程。
- `dialog.html`：补"取消左/确认右"、未保存三按钮、破坏性两种处理。
- `preview.html`：空格预览→翻页→缩放→inspector→关闭整套（当前完全缺失）。

**4. 第 7 层"校验"自动检查项**
- 页面必须含 `.page-header`（标题区存在）。
- 弹窗 `__foot`/`dlg-foot` 中 Cancel 的 DOM 顺序必须在确认之前；不得对 destructive 按钮加 primary class。
- 每视图 prominent/primary 按钮数 ≤ 2；每组 ≤ 1 primary。
- 内容容器 `max-width` 必须命中四档 token 之一（禁止裸数值）。
- 搜索框不得置于页面/sidebar 底部；结果计数不得置底。
- 预览组件必须含关闭方式（Esc + ×）与计数器（多文件时）。
- 标题不得用 app 名；`.nav-title` 文案 <15 字符告警。
