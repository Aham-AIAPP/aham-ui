Aham UI

Aham 来自 aha moment——把灵光一现，做成能用的 AI 工具。每个工具只把一件事做利落。Aham UI 是这一系列工具共享的设计语言：极简、克制、大量留白、内容优先，安静得让内容自己说话。这份文档本身就用 Aham UI 做，所见即规范。

原则

每个 Aham 工具都安静、克制、本地优先。视觉上建立在三条不可妥协的原则上。

1克制高于一切。大量留白；用留白 + 极淡背景层差分隔区域，避免边框。刻意的低信息密度，让内容呼吸。
2蓝是点缀。大面积纯白 + 中性灰；蓝只在 logo、主操作、发送、选中。如果蓝是你第一眼注意到的，就太多了。
3内容优先。层级靠字号、字重、留白，不靠装饰。一个元素不必要就删掉。
反模式（绝不）
✕到处彩色标签 badge（用文字）· 每个数据都挂图标 · 红黄绿仪表灯
✕用背景块/边框分隔区域（用留白）· 渐变、发光、堆叠阴影、毛玻璃装饰
✕卡中卡、框中框 · 一排不等宽按钮 · 蓝色大色块 · 第二个色相 · 任何高饱和色

交付前自检："删掉这个元素 / 颜色 / 图标 / 框，是更干净还是更糟？"——更干净或一样，就删。

图标

默认避免——文字能说清就用文字。内容区用纯点 • + 短文字，不用图标。

✓必要图标：线性、1.5px 描边、单色、无填充，统一一套统一尺寸。仅导航 / 发送栏。
✓单色：继承 currentColor，激活转 ink，仅选中导航 / 主操作转蓝。
✕无 emoji、无多色 / 填充图标、无品牌色文件图标（文件类型用 DOCX 文字 badge）。

在 Office 中应用
